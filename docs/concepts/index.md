# Concepts

These pages explain the shape of Konditional without forcing you to read the
implementation. Use them when you want the mental model behind the quickstart
and the guides.

## What these pages cover

The concepts section tracks the current public behavior of the repo rather
than an aspirational architecture.

- [Namespaces](namespaces.md) explains ownership, identity, and local runtime
  state.
- [Typed features](typed-features.md) covers the supported value shapes and
  what the type parameters mean.
- [Context and axes](context-and-axes.md) explains how runtime inputs are
  modeled.
- [Rules and precedence](rules-and-precedence.md) explains matching and
  ordering.
- [Runtime snapshots](runtime-snapshots.md) explains load, rollback, disable,
  and enable behavior.
- [JSON boundary](json-boundary.md) explains strict decode and typed failures.

## How to use this section

Read the pages in order if you are new to the model. Jump by topic if you are
already using the quickstart and only need to understand one behavior more
deeply.
